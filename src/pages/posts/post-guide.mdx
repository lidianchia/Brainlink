## 文章系统使用指南

本站的文章系统基于 MDX 文件自动化构建。在网站的构建阶段会自动扫描并解析位于 `src/pages/posts` 目录下的所有 MarkdownX (.mdx) 格式文件，并将其转换为对应的网页页面。

MDX 是 Markdown 语法的扩展，集成了 JSX 功能。我们这里主要使用的是 mdx 的 frontmatter 特性来管理文章的元数据。frontmatter 位于文件的最顶部，是由两行`---`包裹的 YAML 格式数据区域。

frontmatter 定义的 Metadata 结构如下：

| Prop        | Status   | Type   | Example                         | Describe                             |
| ----------- | -------- | ------ | ------------------------------- | ------------------------------------ |
| title       | Required | String | title: Example Post             | 文章标题                             |
| description | Required | String | title: Description of the post. | 文章简介                             |
| pubDate     | Required | String | pubDate: 2025-04-29             | 发布日期。格式为YYYY-MM-DD(ISO 8601) |
| modDate     | Optional | String | modDate: 2025-04-29             | 最后修改日期                         |
| pin         | Optional | Number | pin: 2                          | 文章置顶。数字表示指定位序           |

请务必正确填写所有标记为 `Required` 的必填字段，这是文章系统正常工作的必要条件。

以下是一个包含可选字段的完整 frontmatter 示例。只需将这部分内容添加到现有的 .md 文件最顶端即可：

```YAML
---
title: Example Post
description: The description of this article.
pubDate: 2025-04-29
modDate: 2025-05-01
---
```

然后请将文件的扩展名从 `.md` 更改为 `.mdx`。

- 文件命名规范：

文件命名至关重要，因为它将直接映射为文章网页的 URL 路径。请使用简洁且具有描述意义的全小写英文字词(以及数字)，并用连字符(-)连接单词(而不是使用下划线\_)。例如`post-guide.mdx`.

对于大多数有时效性的推送和文章，建议添加日期作为前缀。例如`2025-05-01-today-post.mdx`.

关于文章的图片，请将其放置在 `/assets/img/posts/` 目录下，或使用图床。图片文件命名规则与文章文件相同，并需要添加文章名称作为前缀。例如`2025-05-01-today-post.mdx` 文章的图片请命名并放置于 `/assets/img/posts/2025-05-01-today-post-detailed-description.png`。

有关更详细的 URL 结构推荐阅读 [URL structure best practices for Google](https://developers.google.com/search/docs/crawling-indexing/url-structure).
